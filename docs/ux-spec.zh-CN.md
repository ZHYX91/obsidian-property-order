# Property Order UX 规范

本文描述 Property Order 0.1.1 的当前交互与呈现契约。英文版用于同步阅读；若翻译冲突，以本文为准。

## 属性值拖拽

- 属性值拖拽是桌面应用功能。移动应用不注册属性值拖拽手势处理，并保持 Obsidian 原生长按菜单不变。
- 点击 pill 不启动拖拽；鼠标超过移动阈值后启动，touch/pen 长按后启动。
- 长按阈值前允许原生页面滚动；进入 dragging 后才阻止默认触摸移动，并抑制同一次长按产生的原生值菜单。
- 预览保持源 pill 的渲染尺寸和单行省略，按所属 document 的 visual viewport 缩放与限制，四周保留可见边距。
- drop indicator 必须清楚表达插入槽；同属性 noop、非法跨属性目标和跨文件目标不得写回。
- 完成、取消、冲突、pointercancel、Escape、blur、文件切换或组件消失都清理预览、指示器、cursor class、计时器和临时 listener。
- 内容冲突显示本地化提示并保留最新文件，不自动重试或覆盖。

## 属性键候选

- 增强复用原生菜单节点，不复制一个视觉相似的替代菜单。
- 置顶项在前、普通项居中、置底项在后，隐藏项不占可见导航顺序。
- 键盘选择 class 与最终可见顺序一致；Enter 只激活当前可见项，全部隐藏时不提交。
- 鼠标 hover 后切回键盘时，以最新键盘动作重新建立唯一可见选择。
- Escape、焦点离开、菜单关闭、设置禁用和插件卸载保留或恢复原生关闭与输入行为。
- DOM 结构不匹配、候选文本不可读或宿主选择无法同步时，不留下部分隐藏或部分重排状态。

## 设置页

- General、Value drag、Key order 使用 `tablist`、`tab`、`tabpanel`、`aria-selected` 和 roving `tabindex`。
- Left/Right、Home/End 切换页签；重渲染、旋转和 viewport resize 后，活动页签保持可见并保留合理焦点。
- 精细指针页签高度为 34px，粗指针为 44px；窄宽度下标签栏单行横向滚动，不产生纵向裁切。
- 宽度不超过 480px 时，规则文本框和“添加已有属性”输入纵向占满卡片。
- 保存失败显示 Notice 和 `role="alert"` 未保存状态；Retry 成功后清除提示并执行所需候选刷新。

## 可访问性与已接受边界

- 所有设置控件、页签和错误状态使用可访问名称与语义。
- 键候选支持键盘导航，并保持宿主原生选择行为。
- 桌面属性值重排当前依赖指针输入；缺少键盘直接重排和拖拽 live-region 是已公开限制，不伪装成已完成能力。
- Android 模拟器用于验证原生属性值长按菜单、属性名称候选、旋转、窄屏和生命周期；物理设备的触感、pen 差异和厂商输入栈不在当前证据范围内。
