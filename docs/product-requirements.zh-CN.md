# Property Order 产品需求

本文描述 Property Order 0.1.0 的当前产品边界。英文版用于同步阅读；若实现、测试或翻译与本文冲突，以本文为准。

## 产品目标

Property Order 只增强 Obsidian Properties 的两类顺序：

1. 在当前笔记的顶层 YAML 列表属性中重排值；
2. 按用户规则重排和过滤原生属性键候选。

插件必须保持局部、可撤销和 fail-safe：不能识别宿主 DOM 时保留 Obsidian 原生行为；不能安全解析或验证 frontmatter 时不写入笔记。

## 属性值顺序

- 支持由 Obsidian Properties 渲染为 pill 的顶层 flow list 与 block list。
- 支持同属性重排；启用跨属性拖拽后，支持同一 leaf、同一文件内受支持属性之间移动。
- 属性值拖拽只在桌面应用中运行。鼠标移动达到阈值后开始拖拽；桌面应用收到的 touch/pen 输入在长按后开始拖拽。
- 写回格式为 `preserve`、`flow` 或 `block`。`preserve` 保持当前格式和可保留的标量表示、注释、空行与换行；强制转换只规范化受影响属性。
- 文件、leaf、源/目标内容或 DOM 身份在事务期间变化时取消，不得写入其他文件或覆盖较新内容。
- 冲突、非法结构或不支持结构必须给出诊断并保持磁盘内容不变。

## 属性键候选

- 支持置顶、置底、隐藏通配规则、按名称排序和按使用次数排序。
- 名称排序依次处理数字、拉丁字母、按拼音排列的中文和其他字符；usage 平局使用同一名称比较器。
- 设置页与原生候选菜单必须共享同一排序语义。
- 键盘导航按最终可见 DOM 顺序处理方向键、Home/End、PageUp/PageDown、macOS/iOS Ctrl+P/N 和 Enter。
- 全部候选隐藏时不得提交隐藏项；焦点离开属性名编辑器后不再接管键盘。
- 无法识别 Properties 菜单或同步宿主选择状态时恢复原生顺序、可见性和交互。

## 设置

- 设置使用版本化 schema；未知或非法值归一化，旧版本逐步迁移。
- General、Value drag、Key order 三个页签保留即时生效语义。
- 设置保存失败时保留内存状态，显示本地化 Notice、可访问的未保存提示和重试入口。
- 跨属性拖拽默认关闭；属性键候选增强可独立关闭并完整恢复宿主状态。

## 明确非目标与限制

- 不支持嵌套列表、对象列表、多行 flow sequence、源码模式拖拽或跨文件移动。
- 强制把 block list 转成 flow list 时，允许丢弃只有 block 形式能表达的项目注释和空行。
- 移动应用禁用属性值重排，保留 Obsidian 原生的“编辑 / 从列表中移除 / 复制”长按菜单；属性名称候选排序仍支持移动端。
- 桌面属性值重排只支持指针输入；0.1.0 不提供键盘直接重排命令或屏幕阅读器拖拽 live-region 播报。
- 属性键候选增强依赖可识别的公开 DOM 形态；宿主结构变化时以 fail open 为正确结果。
- 插件只处理 Obsidian 实际暴露为 Properties 的笔记；宿主不呈现 Properties 的 CR-only 文档不承诺真实 UI 写回。
